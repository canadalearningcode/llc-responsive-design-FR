Lequel choisir?

Native

  • Entièrement construites pour leurs plateformes (BlackBerry, iOS, Android, Windows Phone)
  • Dependant de modèle de l'appareil (eg. iOS 5 vs iOS 6)
  • Réseau de distribution aidant au marché (Appstore, Market)
  • Plus grande flexibilité; utilisation des fonctions de l'appareil. ex: Camera/mail
  • Plus grand coût de développement et de temps.

HTML5

  • Simplement un site web - tous les appareils ont un navigateur (tous!)
  • Peu importe le navigateur de l'apparareil, des standards existent
  • Possibilité de modifications en tout temps - les apps natives demandent une mise à jour complète

Combinez Structure et Style

Comment styliser vos éléments HTML avec du CSS?

<p class='my-css-class'>

Vous pouvez styliser tous les éléments ou les différencier avec des classes

p.my-css-class {
  color:blue;
}

Voici un bloc de CSS. Les accolades entourent des propriétés

Exemple

<p>Je suis tout bleu!</p>
<p class='yellow'>Je suis jaune:)</p>
p {
  color:blue;
}
p.yellow {
  color:blue;
  background-color:yellow;
}

Produces:

Je suis tout bleu!

Je suis jaune :)

En CSS, <p class='yellow'> devient p.yellow

Si 'p' est spécifié, le style s'applique à toutes les balises 'p'.

Plus qu'une classes

Vous pouvez donner plus qu'une classe à un élément

<p class='yellow purple'>Je suis mauve et jaune</p>

CSS:

p {
  color:blue;
}
p.yellow {
  background-color:yellow;
}
p.purple {
  color:purple;
}

Je suis mauve et jaune

Notez comment color:blue; a été dépassé

Certains éléments HTML s'alignent naturellement à l'horizontale (inline). Les éléments inline ne supportent généralement pas les propriétés de hauteur, largeur et rembourrage (padding). Les éléments de type block, eux, auront tendance à prendre le plus d'espace possible en largeur.

Display: inline-block;

Les navigateurs sur mobile essayent d'être intelligent avec leurs "zoom to fit screen" (zoom sur le contenu pour l'adapter à la taille de l'écran).

L'appareil essaie d'être intelligent et de vous en montrer le plus possible... mais on ne veut pas ça !

"Zoom to Fit" vs "Zoom to Actual Size"

Zoom to fit screen:Le contenu est ajusté (à 50%), c'est une vue d'oiseau pour voir l'ensemble du contenu.

Zoom to actual size: On voit le contenu à sa taille réelle (100%)

On doit spécifier le comportement voulu au navigateur. vs

Le Viewport

La taille de l'écran de mon navigateur, le ('visual viewport') est de 320px

Nous devons préciser le comportement que nous souhaitons pour le contenu avec 'zoom-to-actual-size' (100%)

Restrict the amount of content (layout viewport) the browser will scale to, to the visual viewport

Encore plus de CSS - Margins et Padding

Margins et Padding (marges et rembourrage) sont utilisés pour créer de l'espace entre entre les éléments et à l'intérieur des éléments.

Voici le 'box model'. Une boîte utilise les marges pour se distancier des autres éléments et utilise le rembourrage pour créer de l'espace entre ses bordures.

Super important pour plus tard

Si la largeur de la boite est définie à 200px, par exemple, et qu'un padding est ajouté, la taille de la boîte sera plus grande.

Donc si nous disons padding:10px;la boîte fera 220px de large. Nous devrons nous en souvenir au moment de l'alignement.

Detail très important: http://www.w3.org/TR/CSS2/box.html

Une Alternative

Récemment, la propriété CSS box-sizing: border-box; (IE8+, any Chrome, Safari 5.1+ (webkit-box-sizing en version antérieure), Opera 8.5+).

Nous permet de considérer le box modeldifféremment l'ajout de padding n'aura plus d'impact sur la largeur/hauteur de l'élément.
